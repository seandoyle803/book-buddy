SUPABASE CLIENT ARCHITECTURE

Create two separate Supabase clients.

1. Browser client

Create:

src/lib/supabase/client.ts

Use createBrowserClient from @supabase/ssr.

This client is for:
- Client Components
- sign up
- sign in
- sign out
- form submissions
- logging reading sessions from the browser
- client-side auth state

It should export a createClient() function.

2. Server client

Create:

src/lib/supabase/server.ts

Use createServerClient from @supabase/ssr.

This client is for:
- Server Components
- server-side data fetching
- dashboard page data
- protected page rendering
- reading authenticated cookies from the request

Wire cookies using cookies() from next/headers.

The server client must read cookies with cookieStore.getAll() and write them back with cookieStore.set().

Both clients must be functions, not global singletons, because server requests need fresh clients.

IMPORTANT DATABASE TRIGGER REQUIREMENT

Use a Supabase auth trigger so profile creation is automatic.

When a user signs up through Supabase Auth:
- Supabase creates a row in auth.users
- a PostgreSQL trigger should automatically create the matching row in public.users
- public.users.supabase_auth_id should equal auth.users.id
- public.users.email should equal auth.users.email
- default daily_goal_minutes should be 15
- default reminder_enabled should be false
- default dark_mode should be true

Do not rely on fragile manual profile insertion after sign-up if the trigger exists.
If the trigger does not exist, provide the SQL needed to create it.

The trigger should be named something like:
on_auth_user_created

The function should create a user profile in public.users.

DATA ACCESS PATTERN

Use the server Supabase client for page-level data fetching.

For example, when /dashboard loads:
1. middleware checks the session
2. dashboard/page.tsx creates a server Supabase client
3. it calls supabase.auth.getUser()
4. it fetches the matching public.users row using supabase_auth_id
5. it fetches the user’s current book
6. it fetches recent reading sessions
7. it calculates weekly activity and progress
8. it renders the page already populated with real data

Use the browser client for user actions such as:
- signing in
- signing up
- starting or ending a reading session
- creating a book
- updating current page
- changing settings
